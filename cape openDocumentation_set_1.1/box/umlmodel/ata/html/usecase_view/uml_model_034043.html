Human Actor Relatioship Use Case Diagram

Human Actor Relatioship (Use Case Diagram)



Flowsheet Builder Actor

A human actor.

The person who sets up the flowsheet, the structure of the flowsheet, chooses thermodynamic models, the unit operation models and solver packages.
This person hands over a working flowsheet to the [Flowsheet User]. The Flowsheet Builder can act as a [Flowsheet User].

Flowsheet User Actor

A human actor.

The person who uses an existing flowsheet. This person will put new data into the flowsheet rather than change the structure of the flowsheet.

Add Stream Use Case

Add Stream Use-Case.

Description:
In order to build a Process Graph, all streams of the flowsheet must be described by giving the initial Unit each stream comes from, as well as the final Unit each stream goes to.
If these units do not exist, they are created. If the unit(s) already exist(s), that is implementation dependent.
The stream has by default a weight (implementation dependent) and properties (To Be Torn = False and Not To Be Torn = False). Use [Add Weight] and [Add Stream Property] to overwrite them.

Assumptions:

Exceptions:

Collaboration Diagram:Initializing Sequence Diagram

Collaboration Diagram:Initializing (Sequence Diagram)



The Collaboration Diagram corresponding to the Sequence Diagram on Initializing GAT.

The [Flowsheet Builder] interacts with the CAPE Simulator through its GUI. The Simulator uses a CO SMST Component in order to perform some Pre-Processor functions.

The [Flowsheet Builder] builds the flowsheet, modifies the flowsheet, displays the modified flowsheet and the [Flowsheet User] launches the Pre-Processor Simulation.

The [Pre-Processor Subsystem] translates these interactions to the CO SMST Component.
It creates a Process Graph from the Flowsheet Factory and it completes it. It creates a Sequential Modular Analysis from the Analysis Factory.
