Download the Flowsheet Server and the First Programs

 

 

You can install on your desktop the Flowsheet Server 5.0 application and the first C++ programs which correspond to the 5 tutorials.

Operating system: Windows 9x, NT.

Disk space requirement: 10.3 MB of free disk space.

 

Installation :

Dowload the 3.2 Mo zipped file (and extract all the files using "folder name" option).
You should have 7 bat files and one folder bin.
The folder bin contains the deployed application files and the idl files while the bat files are the command files which start the application.
Here is the corresponding deployment diagram.

 

Running the application :

  • Starting the Smart Agent : agent.bat
    (Before you attempt to run tutorial programs or Flowsheet Server, you must first start the Smart Agent on at least one host in your local network).

  • Executing Flowsheet Server : FlowSer.bat
    (it displays incomprehensible information according to requests, forget that. It was for debugging)

  • Executing Tutorials : Tutorial1.bat, Tutorial2.bat, Tutorial3.bat, Tutorial4.bat and Tutorial5.bat.

  • Reporting Objects and Services : reporting.bat
    You can run or not this tool at any time. It determines the number of Smart Agent on the network and the exact host on which they are executing. It also reports on all objects that are registered by the Smart Agent (only the global scope objects) and are active on the network. It monitors the status of the network.

 

Closing the application

Close simply the windows but it'd be better if you close the Flowsheet Server window before the Smart Agent window.

 

Uninstallation :

Just delete the 7 bat files and the bin folder.

 

Other Deployment :

The above deployment means that the Flowsheet Server application and the tutorials reside on the same local Windows host.

You could try to locate them on different Windows hosts (within the same local network) : the Flowsheet Server on Windows host A and tutorials on Windows host B.
Don't forget the Trader.txt file (1 Ko - it has the Inter-operable Object Reference) which is generated by the Flowsheet Server. Copy and paste it from host A to host B in your tutorial folder.

 

Notes :

Now you could write your own program and call the Flowsheet Server. Here are its limits (because of some legacy code and the prototype implementation).

  • The identifiers of units need to belong to this interval [1..Number of Units].

  • Streams can't have the same initial unit (iunit) and the same final unit (funit).

  • A stream can't have a funit identical to the iunit.
