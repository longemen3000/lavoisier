Set of Client Programs

 

Easy C++ programs have been developed for the validation step of CAPE-OPEN project. They use services coming from a CAPE-OPEN version 2.0 compliant SMST component (for instance the Flowsheet Server 5.0 application).

Each program performs a numerical analysis on a specific flowsheet: it initialises the ORB, codes the functional block diagram corresponding to the flowsheet, performs the analysis and does release/exit process.

 

Please navigate through the different tutorials : for each tutorial, the numerical analysis performed is indicated, you may access the flowsheet which is analysed as well as the functional block diagram corresponding to the analysed flowsheet, the C++ code is available and the numerical result through the output textual file.

Notes :

  • The M&W Flowsheet is a case study extracted from the Process Flowsheeting 1979 book (§6).
  • The Hexene Flowsheet is extracted from the Process Flowsheeting 1979 book.
    The propylene feed contains 2 mole % propane. In the reactor the propylene is being converted at 20 bar and 200°C to 2-hexene by the reaction :
    2C3H6 --> C6H12
    Conversion of 80% of the propylene is anticipated. The flash separator recovers un-reacted propylene and most of the propane, which acts as an inert substance here. The bleed is to prevent too high a build-up of propane.
  • The Hexene Flowsheet without recycle is similar to the Hexene Flowsheet except that the propylene recycle stream has been suppressed.
  • The ESCAPE'9 Flowsheet was used during the CORBA simulation demonstration which was presented by CAPE-OPEN partners during the ESCAPE'9 congress in Budapest (June 1999). It is using modules supplied by Institut Français du Pétrole.
    This process is dedicated to the catalytic reforming of naphtha cuts. Fresh naphtha feed is injected in the reaction section together with the H2 recycle. The reaction section is mainly composed of a feed / effluent exchanger, 3 furnaces and 3 reactors. The reactor effluent is cooled down and sent to a vapour – liquid phase separator. Liquid effluent is sent to a separation section where LPG and Reformate are recovered as main products (note that this section will not be simulated). Vapour effluent is used as H2 recycle to ensure a sufficient H2 partial pressure in the reactors in order to operate the reforming catalyst as efficiently as possible. Remaining part of the vapour effluent is purged for a later use as H2 rich gas.
