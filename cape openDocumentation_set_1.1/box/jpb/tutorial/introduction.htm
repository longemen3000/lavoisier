Introduction to Tutorials

 

During the design of the CO SMST Component interface, with the aim of validation process some applications were developed for the CORBA architecture. They are fully compliant with the Software Component Documentation Sequential Modular Specific Tools CO-NUMR-IN-08 version 2.0.

  • A C/C++ application called Flowsheet Server which is a CAPE-OPEN compliant SMST component (that corresponds to a "server" side or "soft with a plug"),
  • Some basic C++ programs which use services coming from a CAPE-OPEN compliant SMST component (that corresponds to a "client" side or "soft with a socket"),
  • A complete Java application called Flowsheet Analyser which uses services coming from a CAPE-OPEN compliant SMST component (that corresponds to a "client" side or "soft with a socket").

Note that the Flowsheet Server integrates some Fortran legacy codes. All programs have only a prototype status and have no commercial issue. These applications are deployed, you can download and install them from the Appendices.

 

The development environment was :

  • Windows NT 4.0 SP3,
  • Microsoft DevStudio Visual C++ 5.0 SP3,
  • Java 2 SDK 1.2.1,
  • Rational Rose 98 Professional C++ Edition SP1,
  • Microsoft DevStudio Fortran Powerstation 4.0,
  • Inprise VisiBroker for C++ 3.3 (compliant with the CORBA Specification 2.2),
  • Inprise VisiBroker for Java 3.4 (compliant with the CORBA Specification 2.2).
