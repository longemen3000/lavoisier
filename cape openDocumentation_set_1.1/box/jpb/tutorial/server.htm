Flowsheet Server version 5.0

 

The Flowsheet Server application is a C/C++ application which has been developed for the validation activity within the CAPE-OPEN project. It provides a server to be called by a client for the purpose of analysing a flowsheet. It integrates some Fortran legacy codes from INPT. It is a CAPE-OPEN component compliant with version 2.0 of the SMST Interface Specification.

The skeleton from that component was made from the idl files included in the SMST CO Documentation version 2.0. However, as the CAPE-OPEN Common Services were not fully standardised when the Flowsheet Server was developed, a specific ICapeUtility.idl file has been implemented and used for this application (the implementation of Parameter and Report concepts is simple).

The source codes of implementation objects are not displayed. But you can have a look main program of the Flowsheet Server. It initialises the Object Request Broker (ORB) and Basic Object Adaptator (BOA), creates the ICapeNumericGATComponent object implementation, writes its IOR in the Trader.txt file and waits for incoming request.

This application is deployed, you can download and install it from the Appendices.

 

 

Some technical characteristics about this implementation version :

  • It uses the stringification in order to be ORB independent.
  • This version does not automatically activate servers with the Object Activation Deamon (the Inprise implementation of the implementation repository), and does not use Deferring Object Activation while waiting for a client request.
  • Inheritance implementation is applied (Objects implementation classes inherit from a skeleton class generated by the IDL compiler) instead of a delegation implementation.
  • No persistence capability.
  • Multi-threaded Windows.
