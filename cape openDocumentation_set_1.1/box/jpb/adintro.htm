Introduction

 

 

This part provides the complete Unified Modelling Language (UML) model of the CO SMST Component through the use-case, logical, component and deployment views.

The first step in the UML process is to express the user requirements in the form of a use-case model and some scenario (through sequence and collaboration diagrams). It identifies the «users» of the system, called Actors, and describes, in the form of Use Cases, what they wish the system to do. It also identifies the boundaries of the system. This section formalises the description of the user requirements for Sequential Modular Specific Tools interfacing in the CAPE-OPEN Interface System, as described in Part 2 Textual requirements.

Then, the interface diagrams and the state diagrams describe the design within the logical view.
And finally, the component and deployment views are provided.
