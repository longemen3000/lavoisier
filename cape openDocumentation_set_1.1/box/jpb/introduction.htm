Presentation & Purpose of this Manual

This manual written in full HTML presents the CAPE-OPEN Sequential Modular Specific Tools (SMST) Component which has been developed during the CAPE-OPEN Project in the Work-Package Numerical 4.3 Flowsheet Solvers. It is focused on the Interface Specification, except in the TUTORIAL part where some codes examples are displayed.

The main objective of this manual is to be the reference to :

  • the software suppliers of CAPE-OPEN compliant SMST component ("server" side or "soft with a plug" or "Process Modelling Components"),
  • the software suppliers using a CAPE-OPEN compliant SMST component ("client" side or "soft with a socket" or "Process Modelling Environments").

The content is currently only dedicated to the CORBA architecture. Nevertheless, a lot of thinks would be true for the COM architecture. Also, the global architecture and administration of the SMST Component aren't discussed. Therefore, there will be no details on the "Bind" process and "Exit" process.

Only the specifications proper to the SMST Component are explained. You will find some bliographic references and URL links which refer to :

  • CAPE-OPEN Common Specifications (such as Naming, Error Handling, ...),
  • CAPE-OPEN Common Services (such as Parameter, Reporting, Identification, ...),
  • other CAPE-OPEN Component,
  • or other documents.

The object-oriented technologies which are used in this documentation are based on:

  • the UML 1.1 Documentation,
  • the CORBA/IIOP 2.2 Specifications with the CORBA (C++ and idl to Java) Language Mapping,
  • the CORBAservices (especially LifeCycleServiceSpecification-November 1996),
  • the C++ language,
  • the Java 2 platform.

 

Intended Audience

There is a broad range of activities for people who are interested in CAPE-OPEN, e.g. software experts, process modellers, university researcher workers, engineering managers and IT systems managers.

The identified audience for CAPE-OPEN Documents is as follows:

  • European Commission,
  • Process software developers,
  • Process Modellers who represent the ultimate end-users. They will use CAPE-OPEN based components and simulators for applications,
  • Supply chain and production organizations (e.g. consider their desires to bring together the process models of individual manufacturing units with the overall supply chain considerations, accounting systems, and SAP systems).
  • Simulator system architects and simulation systems designers (particularly those within simulation software suppliers) (e.g. consider their concerns on integrity of applications with options for components mixed from different sources, robustness, reliability, and safety).
  • Managers and administrators of information systems (in all types of organisations, including operating companies, software companies and universities).
  • Organizations responsible for process systems in the process industries. These organizations are concerned with assuring system integrity in a CAPE-OPEN mixed component environment).
  • Concurrent and future standards and technology developers. These include Global CAPE-OPEN (that will follow on from CAPE-OPEN), pdXi, OPC, OPERA and others.
  • Standards organisations (e.g. ISO and AIChE, IChemE, etc).
