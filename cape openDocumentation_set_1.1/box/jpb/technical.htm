Technical Notices

 

Here you could find some technical notes which are relative to the design of the final interface and the idl files.

  • The interface respects the recommendations of the CAPE-OPEN Methods&Tools group. Find its guidelines and conventions in the CO-CD document.
    For instance, the     types definition and errors handling in the ICapeUtility.idl file are the result from the Methods&Tools group work.
  • Don't forget the CAPE-OPEN Common Services inside the CAPE-OPEN Concepts Document.
    Particularly the Parameter concept allows to include some specific items within the interface.
  • By reason of diverse thinks, there is no defined module within the interface and it's a shame.
  • The objects belonging to the interface follow the specifications from the CORBA LifeCycleService (Specification-November 1996).
  • The numerical calculation (perform() method) can't be interrupted by the client side.
  • The interface doesn't support the calculation status concept (to show the state of a running or interrupted numerical calculation).
  • Keep in mind that the final interface diagram is compliant with the CORBA architecture. For the Microsoft COM, it's clear that the design could be (slightly or not) different.
    Somehow the current     idl files are specific to CORBA.
