Background on SMST



Hereafter, you have a quick and basic presentation of solvers at flowsheet level which are integrated in the CO Sequential Modular Specific Tools Component.

Find a good introduction about these functionalities in Process Flowsheeting - A. Westerberg, H. Hutchison, R. Motard, P. Winter - Cambridge University Press - 1979. especially in the chapter 6 The Sequential Modular Approach to flowsheeting.


This manual concerns only Sequential Modular simulators which are the most commonly used for process simulation.

In this approach, there are, at least, two levels of calculations :
the unit level : calculation of all the output stream values given values for all input streams and for all operating and equipment parameters
the flowsheet level : calculation of all the tear stream values given values for all residuals on recycle streams and specification equations.
Note that, at least, streams include material, information and energy streams.

The CO SMST Component is only concerned by the flowsheet level : partitioning of the flowsheet, search of the calculation sequence of the units.The different tools are the following : Partitioning Ordering Tearing Sequencing Full Sequential Modular Analysis

These functionalities are called by the Pre-Processor part of a CAPE Simulator. They are integrated into the CO SMST Component.

Partitioning

Partitioning a flowsheet is to locate the groups of units (single unit and Maximum Cyclic Networks MCN) which must be solved together. Each of these groups should contain the smallest possible number of units. There is one unique result for the partitioning of a given flowsheet.

MCN = a cycle which is not contained into another cycle (usually called Partition)
A Partition can be a group of units or a single unit.

Ordering

Ordering a flowsheet is to place the Partitions in a proper sequence for computation.
There can be several solutions for ordering a flowsheet.

Tearing

Tearing is the analysis of each Partition to determine tear streams. For each Partition, a set of independent variables defining the tear streams will be used as iterative variables at the flowsheet level.

There are many competing algorithms which give significant differences in performance.
Hereafter some bibliographic references:
Barkley and Motard (1972)
Pho and Lapidus (1973)
Upadhye and Grens (1975)
Pibouleau (1980)
Motard and Westerberg (1981)


The result and the efficiency of the tearing depend on the criteria chosen by developers of tearing algorithms. The final choice of tear streams is essential for the convergence capabilities.

The criteria are the following :
the minimum number of tear streams
assign a weight to each stream. This weight can reflect the expected difficulty associated with tearing the stream and may, for example, be equal to the number of significant torn variable values in each stream. Search the tear streams which minimise the sum of the weights
tear to give the best convergence characteristics. For example, Upadhye and Grens define the decomposition of a flowsheet, that is the choice of tear streams, as belonging to two classes : redundant and non redundant. Non redundant decomposition have better convergence properties than redundant ones for the method of successive substitution
others

These above criteria are just mentioned in order to illustrate the strategy that can be used. CAPE-OPEN project makes no judgement about their relative merits, it must only provide all the useful information to tearing developers through the interface definition.

Sequencing

For each Partition where tear streams are determined, search for the computational order of units.

Full Sequential Modular Analysis

This analysis corresponds to a series of the previous analysis. It allows to do directly the full sequential modular analysis of a flowsheet e.g. search of the calculation sequence of the partitions and units.
So, that means it calculates the Partitioning of the flowsheet, next the Ordering of each partitions, next the Tearing of each partitions and next the Sequencing of units of each partitions.
